## 3. 数据库初始化

### 3.1 创建数据库

```sql
CREATE DATABASE IF NOT EXISTS `video`
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```

### 3.2 执行初始化 SQL

```bash
mysql -u root -p video < sql/vidsiilo-init.sql
```

初始化脚本将创建以下表：

| 表名 | 说明 |
|------|------|
| `video_info` | 视频解析信息缓存（TTL 1 小时） |
| `download_task` | 下载任务全生命周期记录 |
| `system_config` | 运行时可修改的系统配置 |
| `sys_user` | 系统用户表 |
| `sys_token` | 登录 Token 持久化 |
| `parse_rate_limit` | 未登录用户解析次数限流 |

## 4. 方式一：Docker Compose 部署（推荐）

### 4.1 克隆代码

```bash
git clone <仓库地址>
cd media-hub
```

### 4.2 配置环境变量

```bash
vim .env
```

编辑 `.env`：

```dotenv
# 后端对外端口（默认 8080）
BACKEND_PORT=8080

# 可选：需要登录认证的站点才需要配置 cookies
# COOKIES_FILE=/path/to/cookies.txt
```

### 4.3 配置数据库连接

编辑 `media-hub-server/src/main/resources/application-prod.yml`，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/video?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: <你的数据库密码>
```

> **注意**：生产环境请务必修改数据库密码，不要使用默认值。

### 4.4 构建并启动

```bash
# 构建镜像并后台启动
docker compose up -d --build

# 查看启动日志
docker compose logs -f
```

### 4.5 初始化数据库

容器启动后，执行数据库初始化：

```bash
# 若 MySQL 运行在宿主机
mysql -u root -p video < sql/vidsiilo-init.sql

# 若 MySQL 也运行在 Docker 容器中
docker exec -i <mysql容器名> mysql -u root -p video < sql/vidsiilo-init.sql
```

### 4.6 验证

```bash
curl http://localhost:8080/actuator/health
# 期望响应：{"status":"UP"}
```

## 5. 方式二：手动部署（裸机）

### 5.1 安装 JDK 17

```bash
# Ubuntu/Debian
apt-get install -y openjdk-17-jre

# CentOS/RHEL
yum install -y java-17-openjdk

# 验证
java -version
```

### 5.2 安装 ffmpeg

```bash
# Ubuntu/Debian
apt-get install -y ffmpeg

# CentOS（需 EPEL）
yum install -y epel-release && yum install -y ffmpeg

# 验证
ffmpeg -version
```

### 5.3 安装 yt-dlp

```bash
curl -fsSL https://github.com/yt-dlp/yt-dlp/releases/latest/download/yt-dlp \
    -o /usr/local/bin/yt-dlp
chmod a+rx /usr/local/bin/yt-dlp

# 验证
yt-dlp --version
```

### 5.4 安装 Python 及 opencv

```bash
# 创建虚拟环境（推荐，与系统 Python 隔离）
python3 -m venv /opt/mediahub/venv
/opt/mediahub/venv/bin/pip install --no-cache-dir opencv-python-headless

# 验证
/opt/mediahub/venv/bin/python -c "import cv2; print(cv2.__version__)"
```

### 5.5 构建 JAR

```bash
# 需要 Maven 3.6+ 和 JDK 17
mvn package -DskipTests

# 产物路径
ls media-hub-server/target/media-hub-server-1.0.0.jar
```

### 5.6 准备目录

```bash
# 下载文件存储目录
mkdir -p /home/video/downloads
```

### 5.7 修改配置

编辑 `media-hub-server/src/main/resources/application-prod.yml`，按实际环境修改：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/video?...
    username: root
    password: <数据库密码>

app:
  inpaint:
    python-path: /opt/mediahub/venv/bin/python   # 虚拟环境 Python 路径
  ytdlp:
    path: /usr/local/bin/yt-dlp
  download:
    dir: /home/video/downloads
```

### 5.8 启动应用

```bash
java -Xmx2048m -Xms1024m \
    -jar media-hub-server/target/media-hub-server-1.0.0.jar \
    --spring.profiles.active=prod
```

#### 使用 systemd 管理进程（推荐生产环境）

创建 `/etc/systemd/system/media-hub.service`：

```ini
[Unit]
Description=MediaHub Backend
After=network.target mysql.service

[Service]
User=www-data
ExecStart=/usr/bin/java -Xmx2048m -Xms1024m \
    -jar /opt/mediahub/media-hub-server-1.0.0.jar \
    --spring.profiles.active=prod
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable media-hub
systemctl start media-hub
systemctl status media-hub
```

## 6. 配置项说明

### 6.1 application.yml 关键配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `server.port` | `8080` | HTTP 监听端口 |
| `spring.datasource.url` | `jdbc:mysql://127.0.0.1:3306/video` | MySQL 连接串 |
| `spring.datasource.username` | `root` | 数据库用户名 |
| `spring.datasource.password` | — | 数据库密码（生产必须修改） |
| `spring.servlet.multipart.max-file-size` | `2GB` | 单文件上传上限 |

### 6.2 app 业务配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `app.ytdlp.path` | `/usr/local/bin/yt-dlp` | yt-dlp 可执行文件路径 |
| `app.ytdlp.cookies-file` | `/home/video/cookies.txt` | cookies 文件路径（可选） |
| `app.download.dir` | `/home/video/downloads` | 下载文件存储目录 |
| `app.download.max-concurrent` | `5` | 最大并发下载数 |
| `app.download.queue-capacity` | `10` | 等待队列容量，超出拒绝新任务 |
| `app.cleanup.file-retain-hours` | `2` | 完成/失败任务文件保留时长（小时） |
| `app.cleanup.scan-interval-ms` | `600000` | 文件清理扫描间隔（毫秒） |
| `app.rate-limit.parse-per-ip-per-minute` | `10` | 每 IP 每分钟最多解析次数 |
| `app.cache.video-info-ttl-minutes` | `60` | 视频信息缓存有效期（分钟） |
| `app.auth.token-expire-hours` | `48` | Token 有效期（小时） |
| `app.inpaint.python-path` | `/usr/bin/python3` | Python 可执行路径（prod 用 venv） |

### 6.3 system_config 运行时配置

以下配置存储在数据库 `system_config` 表，**无需重启**即可生效：

| config_key | 默认值 | 说明 |
|------------|--------|------|
| `rate_limit.parse_per_ip_per_minute` | `10` | 每 IP 每分钟解析次数上限 |
| `rate_limit.download_per_ip_per_day` | `20` | 每 IP 每天下载次数上限 |
| `download.max_concurrent_tasks` | `5` | 全局最大并发下载数 |
| `download.queue_max_size` | `10` | 下载队列最大容量 |
| `cache.video_info_ttl_minutes` | `60` | 视频信息缓存时效（分钟） |
| `cleanup.file_retain_hours` | `2` | 文件保留时长（小时） |
| `cleanup.scan_interval_minutes` | `10` | 清理任务扫描间隔（分钟） |
| `download.max_file_size_gb` | `2` | 单次下载最大文件大小（GB） |
| `download.socket_timeout_seconds` | `60` | yt-dlp 网络超时（秒） |
| `download.retries` | `3` | yt-dlp 失败重试次数 |

## 7. 可选：LaMa AI 水印修复增强

LaMa 模型对复杂纹理的水印修复效果显著优于 OpenCV，未安装时会自动降级。

### 安装 LaMa 依赖

```bash
# 激活虚拟环境
source /opt/mediahub/venv/bin/activate

# 安装（需要约 2-4 GB 磁盘空间，含 PyTorch）
pip install simple-lama-inpainting torch --index-url https://download.pytorch.org/whl/cpu

# 验证
python -c "from simple_lama_inpainting import SimpleLama; print('LaMa OK')"
```

> **提示**：首次运行 LaMa 时会自动下载预训练模型权重（约 200MB），请确保服务器可访问外网或提前下载到缓存目录。

## 8. 可选：yt-dlp Cookies 配置

部分视频平台需要登录状态，可通过 cookies 文件提供：

1. 使用浏览器插件（如 [Get cookies.txt LOCALLY](https://chromewebstore.google.com/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc)）导出 cookies 为 Netscape 格式
2. 将文件放置到服务器：

```bash
cp cookies.txt /home/video/cookies.txt
chmod 600 /home/video/cookies.txt
```

3. 确认配置文件中路径一致：

```yaml
app:
  ytdlp:
    cookies-file: /home/video/cookies.txt
```

## 10. 常见问题

### yt-dlp 下载失败 / 解析超时

- 检查网络连通性，yt-dlp 需要访问目标视频站点
- 部分平台需要 cookies，参考 [第 8 节](#8-可选yt-dlp-cookies-配置)
- 升级 yt-dlp：`yt-dlp -U`

### 水印修复返回 500

- 确认 Python 路径配置正确（`app.inpaint.python-path`）
- 确认 `opencv-python-headless` 已安装：
  ```bash
  /opt/mediahub/venv/bin/python -c "import cv2"
  ```

### 数据库连接失败

- 确认 MySQL 服务已启动：`systemctl status mysql`
- 确认数据库 `video` 存在且用户有权限
- 检查 `application-prod.yml` 中连接串、用户名、密码是否正确

### 端口被占用

```bash
# 查看占用 8080 端口的进程
lsof -i :8080
# 修改端口：在 application.yml 中将 server.port 改为其他值
```
